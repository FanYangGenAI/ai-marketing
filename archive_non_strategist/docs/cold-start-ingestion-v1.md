# 冷启动：多素材入库与产品理解 — 方案初稿 v1

> 状态：初稿  
> 日期：2026-03-22  
> 关联：`docs/system-design-draft.md`（v1.3）、Asset Library、LessonMemory / 产品级记忆设想

---

## 1. 背景与目标

冷启动阶段，用户在创建产品项目时除 **PRD / 文本文档** 外，还可上传 **图片类素材**（如 Logo、产品截图、营销配图等）。系统需要具备 **消化与理解** 能力，将内容分流为：

- **可复用媒体资产**：进入素材库，供后续 Director / Creator 直接引用或再加工。
- **长期产品认知**：从文本与图片中提炼结构化知识，形成对产品的稳定理解，供 Strategist、Planner、Scriptwriter 等在后续营销物料生成中注入上下文。

本初稿在先前讨论基础上，根据当前决策 **收窄范围** 并补充 **前端素材库编辑** 要求。

---

## 2. 范围与边界

### 2.1 本期支持的媒体类型

| 类型 | 本期 | 说明 |
|------|------|------|
| 文本 | ✅ | PRD、补充说明、Markdown / 纯文本等（具体格式以实现为准） |
| 图片 | ✅ | Logo、截图、静态营销图等（格式与大小上限在实现阶段定义） |
| 视频 / GIF 动图等 | ❌ 不支持上传 | 理解成本高，**后续版本扩展**；文档与 UI 中明确提示「暂不支持」 |

### 2.2 版本策略

不做复杂的版本分支体系：以 **「当前生效的一套素材与认知」** 为主即可。用户通过 **替换上传** 与 **素材库内编辑（含删除）** 维护现状；若后续需要轻量历史追溯，可在实现期增加「删除软删除 / 操作日志」等，**不作为本期必做**。

### 2.3 素材库前端编辑（本期必含）

用户在 **前端素材库视图** 中可对已入库素材进行：

- **删除**：从素材库移除该条记录；若底层文件策略为「仅标记删除」或「物理删除」，由实现阶段选定（建议至少 **对用户不可见**，并避免被流水线误引用）。
- **修改备注**：编辑素材的说明文字（如用途、拍摄场景、是否可作首图等），供检索与 Agent 上下文使用。

---

## 3. 用户旅程（冷启动）

1. **创建项目**：项目名称、平台、user_brief（与现有产品设计对齐）。
2. **上传**：提交 PRD（或主文档）+ 可选多个图片文件；可选 **简单分类标签**（如 brand / product_ui / marketing_ref，具体枚举以实现为准）。
3. **异步理解任务**：系统对文本做结构化提炼，对图片做说明与标签；展示进度（可与现有流水线状态展示思路一致）。
4. **审阅与确认（可选但推荐）**：展示「产品档案摘要草案」「素材列表」「文本与图像推断冲突（若有）」；用户确认或微调后 **生效**。
5. **日常使用**：进入每日 Pipeline 时，自动注入已确认的产品认知摘要 + 素材索引；用户在素材库中持续 **删改备注** 以纠偏。

---

## 4. 系统能力（逻辑模块）

| 模块 | 职责 |
|------|------|
| **Ingestion** | 接收上传、类型与白名单校验、大小限制、存储路径与 hash、写入清单（manifest）。 |
| **Understanding** | 文本：结构化抽取与摘要；图片：caption / 标签 / 与 PRD 的粗一致性提示（实现可选用 vision 模型）。 |
| **Knowledge Compiler** | 将理解结果编译为固定 schema 的 **产品档案**（便于程序读取与后续扩展字段）。 |
| **Asset Registrar** | 注册素材库条目：文件引用、hash、类型、备注、标签、是否启用；供 Director 等查询。 |
| **Memory Writer** | 将「非文件本体」的结论写入 **长期产品认知** 存储（与投放反馈类的 LessonMemory 建议 **命名空间分离**，避免混淆）。 |
| **Retrieval** | 为 Strategist / Planner 等提供统一入口：拉取「产品档案摘要 + 高优先级素材说明 + 用户备注」。 |

---

## 5. 数据与落盘（概念）

以下为概念层约定，具体文件名与目录可与现有 `campaigns/{product}/` 布局对齐或由 `server` 统一托管。

- **原始附件目录**：仅存文本与图片原文，便于审计与重跑理解任务。
- **`attachment_manifest`（或等价物）**：每个文件的 id、类型、hash、用户标签、处理状态、备注（备注可被用户在素材库中修改并回写）。
- **产品档案**：人类可读 + 机器可读一份（如 YAML/JSON），包含定位、受众、卖点结构、创作禁忌、术语等；**来源**尽量指向具体文件或摘录，降低幻觉。
- **素材库索引**：与现有 Asset Library 设计衔接；每条记录含 **备注字段**，与前端编辑对应。

冲突策略：当文本与图片推断不一致时，写入档案时 **标注待确认**，不静默覆盖用户 PRD 中的明确陈述。

---

## 6. 与现有架构的衔接

- **Strategist（冷启动路径）**：除行业与竞品类输入外，强制合并 **产品档案压缩摘要** 与 **标记为「品牌必填」类素材**（如 Logo）的引用说明。
- **Director**：优先从素材库选取用户已提供截图 / Logo，减少不必要的生成。
- **Audit**：可增加与「产品事实来源」相关的检查思路（细则在 Audit 清单迭代中落地）。

---

## 7. 前端能力小结

| 能力 | 说明 |
|------|------|
| 上传 | 文本 + 图片；明确提示不支持视频类。 |
| 冷启动任务进度 | 解析 / 理解中状态展示。 |
| 素材库列表 | 缩略图、类型、标签、备注。 |
| 编辑 | **修改备注**、**删除** 条目。 |
| （可选）确认页 | 产品档案草案与冲突提示。 |

---

## 8. 分阶段建议

| 阶段 | 内容 |
|------|------|
| **MVP** | 文本 + 图片上传；manifest + 素材库注册；产品档案初稿；Strategist / Pipeline 可消费摘要；素材库 **备注编辑 + 删除**。 |
| **后续** | 视频 / GIF；增量理解；软删除与操作日志；更细的质量评分与 Audit 规则。 |

---

## 9. 待定与风险

- 图片数量与总大小上限、单文件上限；vision 调用成本与批处理策略。
- 「删除」是硬删除还是禁用位，需与 Asset Library 查重及历史 campaign 引用约定一致。
- 用户上传截图中的隐私与打码策略（可与现有 privacy-mask Skill 策略对齐）。
