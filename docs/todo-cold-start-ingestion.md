# TODO List — 冷启动：多素材入库与产品理解（v1）

> 方案文档：`docs/cold-start-ingestion-v1.md`  
> 创建日期：2026-03-22  
> 范围：**文本 + 图片**；**不支持视频/GIF 上传**（后续扩展）  
> 目标：冷启动上传 → 理解 → 素材库 + 产品档案 + 长期认知；前端素材库 **备注编辑 / 删除**；Pipeline / Strategist 可消费摘要

---

## 阶段 0：规格与决策（阻塞后续实现）

- [ ] 定稿 **允许的图片 MIME/扩展名**、**单文件大小上限**、**单次上传数量/总大小上限**
- [ ] 定稿 **用户可选分类标签** 枚举（如 `brand` / `product_ui` / `marketing_ref`）及默认值
- [ ] 定稿 **删除策略**：硬删除 vs `disabled` / `deleted_at` 软标记（需与 Asset Library hash 查重、历史日录引用一致）
- [ ] 定稿 **产品认知存储**：独立 `product_memory` 文件 vs 与现有 `memory/` 分 namespace；路径与 schema 草图写入方案文档附录（可选）

---

## 阶段 1：数据模型与落盘

- [ ] 设计并实现 **`attachment_manifest`**（或等价物）：`id`, `type`, `path`, `hash`, `user_tags`, `note`, `status`, `created_at` 等
- [ ] 约定 **原始附件目录**（如 `campaigns/{product}/attachments/`）与 **只接受文本+图片** 的校验逻辑
- [ ] 设计并实现 **产品档案** 文件（JSON/YAML）：定位、受众、卖点、禁忌、术语、**来源引用**、**待确认冲突** 字段
- [ ] 与现有 **Asset Library**（`index.json` 等）对齐：新上传图片注册为可引用资产，并带 **备注** 字段
- [ ] **Memory Writer**：将编译后的非文件知识写入长期产品认知存储（与 LessonMemory 分离）

---

## 阶段 2：后端 — Ingestion & API

- [ ] **multipart 上传接口**：PRD/文本 + 多图片；拒绝视频等非白名单类型；错误信息明确
- [ ] 上传后 **写入 manifest**、落盘文件、计算 **hash**、更新 Asset Library 索引（若适用）
- [ ] **GET** 素材列表（供前端素材库）：含缩略图 URL 或文件 id、标签、备注、状态
- [ ] **PATCH** 单条素材：**修改备注**（校验长度与 XSS/注入安全按现有栈处理）
- [ ] **DELETE** 单条素材：按阶段 0 决策执行移除或标记禁用，并保证流水线不再选中
- [ ] （可选）**POST** 触发/查询 **冷启动理解任务** 状态（与现有 run/status 模式一致亦可）

---

## 阶段 3：Understanding / Knowledge Compiler

- [ ] **文本管道**：读取 PRD/补充文本 → 结构化抽取 → 写入产品档案草案
- [ ] **图片管道**：对每张图生成 caption/标签；可选与 PRD 做 **粗一致性** 检测
- [ ] **Knowledge Compiler**：合并文本与图片理解结果 → 更新产品档案；冲突写入 **待确认** 区块
- [ ] **成本控制**：批处理、可配置模型、失败重试与单图超时策略（与 `llm_config` 协调）
- [ ] 任务 **异步化** + 进度字段（供前端轮询或 WebSocket 后续再说）

---

## 阶段 4：前端

- [ ] 创建项目 / 项目设置页：**文本 + 多图上传**；**显式提示「不支持视频」**
- [ ] **冷启动理解进度** UI（解析中 / 完成 / 失败原因）
- [ ] **素材库视图**：列表或网格、缩略图、类型、标签、备注展示
- [ ] 素材 **备注编辑**（inline 或弹窗）→ 调用 PATCH
- [ ] 素材 **删除** → 确认对话框 → 调用 DELETE
- [ ] （推荐）**审阅页**：产品档案摘要草案 + 冲突列表 +「确认生效」按钮

---

## 阶段 5：与 Agent / Pipeline 衔接

- [ ] **Retrieval** 辅助模块（或函数）：组装「产品档案摘要 + 高优先级素材说明 + 用户备注」供注入 context
- [ ] **Strategist**：冷启动路径下合并上述摘要与品牌类素材（如 Logo）说明
- [ ] **Planner / Scriptwriter**（按需）：上下文注入产品档案要点（避免重复造轮子）
- [ ] **Director**：优先查询素材库中用户提供的截图/Logo，再决定生成任务
- [ ] **Audit**（可选 MVP 后）：`audit_checklist` 增加与「产品事实来源」相关的检查项草案
