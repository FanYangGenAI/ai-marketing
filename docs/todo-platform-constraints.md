## 阶段 4：统一校验器（确定性）

- [ ] 新增文案校验模块（建议路径：`src/orchestrator/content_validator.py`）
- [ ] 输入：`platform_rules` + `title` + `body`；输出：`ok` + `violations[]`
- [ ] 违规码实现：`TITLE_MISSING`、`TITLE_TOO_LONG`、`BODY_MISSING`、`BODY_TOO_LONG`
- [ ] 违规详情包含：`code`、`actual`、`limit`、`message`
- [ ] 单测覆盖边界：恰好 20 / 1000、空字符串、仅空白、超 1 字等

## 阶段 5：Agent 注入与执行闭环

- [ ] 校验失败时写入标准化结果并路由 `Reviser` 定向回修
- [ ] Audit 报告中展示违规码与修复建议，保证可解释性
